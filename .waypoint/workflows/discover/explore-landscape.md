## Step 2: Explore Landscape

### Your Objective
Build a comprehensive view of the landscape before diving deep.

### For Brownfield (Existing Code)
- Review directory structure
- Identify major components/modules
- Find entry points (main, index, routes)
- Note dependencies (package.json, requirements.txt)
- Check for existing documentation

### For Greenfield (New Project)
- Map the problem domain
- Identify key entities/concepts
- Research similar solutions
- List stakeholders and their needs
- Identify integration points

### Output Format
```
## Landscape Overview

### Key Components
- [Component 1]: [Purpose]
- [Component 2]: [Purpose]

### Dependencies
- [Dependency]: [Why it's used]

### Entry Points
- [File/Route]: [What it does]
```
